# GroupDocs.Comparison API 教學與開發者指南

![GroupDocs.Comparison 橫幅](./groupdocs-comparison-net.svg)

歡迎閱讀 **完整的文件比較指南**，使用 **GroupDocs.Comparison API**！我們的完整教學將示範如何在各種格式（包括 **Word、PDF、Excel、PowerPoint、圖片等**）中高效找出文件差異。無論您是建立 .NET 網路服務或 Java 桌面應用程式，本指南都會提供快速整合強大文件比較功能的實作步驟。

## 快速解答
- **GroupDocs.Comparison API 的功能是什麼？** 它會偵測並標示兩份相同或不同格式文件之間的變更。  
- **支援哪些平台？** .NET（Framework、.NET Core、.NET 5/6）與 Java（8 以上）。  
- **開發時需要授權嗎？** 免費試用可用於評估；正式上線需購買商業授權。  
- **可以比較受密碼保護的檔案嗎？** 可以 – API 允許在開啟受保護文件時提供密碼。  
- **有產生視覺預覽的方式嗎？** 當然，API 能產生並排或疊加的比較結果預覽圖。

## 什麼是 GroupDocs.Comparison API？
**GroupDocs.Comparison API** 是一組函式庫，讓開發者能以程式方式比較文件的內容、版面與格式。它支援超過 100 種檔案類型，提供詳細的變更日誌，並允許透過程式碼接受或拒絕修改。

## 為什麼使用 GroupDocs.Comparison API？
- **多格式支援** – 在不需先轉檔的情況下比較 Word、PDF、Excel、PowerPoint、圖片、電郵等多種檔案。  
- **豐富的變更偵測** – 自動標示插入、刪除、格式調整與樣式變更。  
- **程式化變更管理** – 在工作流程中接受或拒絕特定變更，適合審核系統。  
- **安全處理** – 安全地處理加密或受密碼保護的文件。  
- **高效能** – 最佳化演算法能有效處理大型檔案與批次資料夾比較。

## 為什麼使用 GroupDocs.Comparison？
- **多格式支援** - 比較不同格式的文件  
- **詳細變更偵測** - 識別插入、刪除、格式變更與樣式修改  
- **變更管理** - 以程式方式接受或拒絕特定變更  
- **安全比較** - 處理受密碼保護的文件  
- **高效能** - 使用最佳化演算法有效比較大型文件

## 支援的文件格式

GroupDocs.Comparison 支援多種文件格式：

| 類別 | 格式 |
|----------|---------|
| **文字處理** | DOCX, DOC, ODT, RTF, TXT |
| **試算表** | XLSX, XLS, ODS, CSV |
| **簡報** | PPTX, PPT, ODP |
| **PDF 文件** | PDF, PDF/A |
| **圖片** | JPG, PNG, BMP, GIF, TIFF |
| **電郵** | EML, MSG |
| **以及更多...** | HTML, EPUB, DJVU |

## 常見問題

**Q: 我可以在商業產品中使用 GroupDocs.Comparison API 嗎？**  
A: 可以，正式上線需購買有效的商業授權。免費試用僅供評估使用。

**Q: API 支援受密碼保護的檔案嗎？**  
A: 完全支援。載入來源文件時可提供文件密碼。

**Q: 支援哪些 .NET 版本？**  
A: API 相容於 .NET Framework 4.5 以上、.NET Core 3.1 以上、.NET 5 以及 .NET 6+。

**Q: API 如何處理大型文件或批次資料夾比較？**  
A: 透過串流與最佳化演算法降低記憶體使用，並可使用資料夾比較功能一次比對整個目錄。

**Q: 能否自訂比較輸出的視覺樣式？**  
A: 可以，Comparison Options 允許您定義顏色、標記樣式與輸出格式，以產生符合需求的差異圖。
